## 一、项目整体架构建议

### 当前问题

单页线性体验，用户看完就结束，缺乏深度探索空间。

### 建议架构：多章节叙事结构

```
第一章：序章（现有开场动画）
    ├── 视频帧滚动动画
    ├── AI 诗句生成
    └── 引导"开启探索"
            ↓
第二章：中轴线长卷（新功能）
    ├── 午门 → 太和殿 → 乾清宫 → 御花园 → 神武门
    ├── 视差滚动效果
    └── 建筑信息卡片
            ↓
第三章：匠心解构（新功能）
    ├── 榫卯结构 3D 爆炸图
    ├── 斗拱结构拆解
    └── 用户可交互旋转
            ↓
第四章：数字典藏
    ├── 文物 3D 展示
    └── 纹样图案库
            ↓
尾声：传承
    └── 团队介绍、技术栈、致谢
```

```
│  中轴线（快速滚动）                     │
│  ═══════════════════════════════════   │
├────────────────────────────────────────┤
│  地面层（慢速滚动）                     │
│  🌳  🌳  🌳  🌳  🌳  🌳  🌳  🌳       │
└────────────────────────────────────────┘
```

**代码示例：**

```javascript
const landmarks = [

  trigger: '#scroll-container',
```

```jsx

function MortiseTenon() {
  const [exploded, setExploded] = useState(false)
```

### 1. 沉浸式声音设计

**为什么重要：** 声音是营造沉浸感的关键，大多数参赛作品都忽略了这一点。

**声音层次：**

```
第一层：环境音（持续）
├── 晨钟暮鼓（根据时间变化）
├── 风声穿过宫殿
└── 远处宫人脚步声

第二层：交互音效（触发）
├── 滚动时的"卷轴展开"声
├── 点击建筑的"木门开启"声
└── 切换章节的"编钟"过渡音

第三层：叙事旁白（可选）
└── AI 生成的建筑故事讲解
```

**技术实现：**

```javascript
const audioContext = new AudioContext();
const panner = audioContext.createPanner();
panner.panningModel = 'HRTF'; // 3D 空间音效

scrollTrigger.onUpdate((progress) => {
  panner.positionX.value = progress * 100;
});
```

## 七、给大一学生的建议

1. **不要一次性做太多**
   - 先完成一个功能，测试通过后再做下一个
   - "能跑起来"比"完美"更重要

2. **善用现成资源**
   - 3D 模型：Sketchfab、Turbosquid 有免费的中国古建筑模型
   - 素材图片：故宫博物院官网有高清图片资源
   - 代码参考：GitHub 搜索 "three.js architecture"

3. **版本控制**
   ```bash
   git add .
   git commit -m "feat: 完成中轴线长卷基础结构"
   ```

4. **模块化开发**
   - 每个章节独立开发，互不影响
   - 第一章（已完成）→ 不动它
   - 第二章（新开发）→ 新建文件
